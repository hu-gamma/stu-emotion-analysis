# 消融实验报告：V6a (Baseline) vs V6b-Cleaned

**对比模型**: 
- V6a: 原始 split_train.csv + dropout=0.1
- V6b-Cleaned: 清洗后 split_train_cleaned.csv + dropout=0.3 (增强正则化)

**测试集**: split_test.csv (9951条, 8类)

---

## 1. 整体性能对比

| 指标 | V6a (Baseline) | V6b-Cleaned | 变化 |
|------|----------------|-------------|------|
| **测试集 Accuracy** | 66.59% | 66.56% | -0.03% |
| **测试集 Macro F1** | **0.5703** | 0.5623 | -0.0080 |
| **测试集 Weighted F1** | **0.6636** | 0.6627 | -0.0009 |
| **训练集 Accuracy** | 84.50% | **90.09%** | **+5.59%** |
| **训练集 Macro F1** | 0.8677 | **0.9127** | **+0.0450** |
| **Train-Test Accuracy Gap** | 17.91% | **23.53%** | **+5.62%** |
| **Train-Test Macro F1 Gap** | 0.2974 | **0.3504** | **+0.0530** |
| **Silhouette Score** | 0.0427 | **0.0846** | **+0.0419** |
| **最佳 Val Macro F1** | - | **0.5808** | - |

### 关键发现

- **测试性能几乎持平**：V6a 和 V6b-Cleaned 在测试集上表现几乎相同（Macro F1 差距 < 0.01）
- **但过拟合显著加剧**：V6b-Cleaned 的 Train-Test F1 gap 从 0.297 → 0.350（+18%）
- **特征空间分离度提升**：Silhouette Score 从 0.043 → 0.085（+98%）
- **验证集表现更好**：V6b-Cleaned 最佳 Val Macro F1 = 0.5808，高于 V6a 测试集 0.5703

### 解读

清洗数据集 + 增强 Dropout 的组合效果复杂：
1. ** dropout=0.3 有效提升了特征空间分离度**（Silhouette +98%）
2. **但清洗后的数据让模型更容易记忆**（训练 F1 从 0.87 → 0.91）
3. **测试性能没有提升**，说明清洗去除的 52 条样本中可能包含了对泛化有帮助的困难样本

## 6. 综合评估

### V6b-Cleaned 相比 V6a 的改进
1. **验证集性能更好**：最佳 Val Macro F1 = 0.5808 vs V6a 测试集 0.5703
2. **特征空间分离度翻倍**：Silhouette 0.085 vs 0.043
3. **大类别（悲伤/快乐）测试 F1 略有提升**
4. **悲伤/快乐召回率提升**：分别 +2.0% 和 +3.5%

### V6b-Cleaned 的问题
1. **过拟合显著加剧**：Train-Test F1 gap 从 0.297 → 0.350
2. **中等样本类别性能下降**：愤怒/厌恶/好奇 测试 F1 下降
3. **"悲伤吸星"效应未解决**：愤怒→悲伤 错分率 32.4%（比 V6a 的 31.1% 还高）
4. **清洗数据可能去除了困难样本**：导致训练集更易拟合但泛化未提升

### 关键洞察

> **清洗数据集 + dropout=0.3 的组合效果复杂**：
> - 正面：验证集性能提升、特征空间分离度改善、大类别召回率提升
> - 负面：过拟合加剧、中等样本类别性能下降
> 
> 这说明**仅靠数据清洗和 dropout 增强不足以解决核心问题**。清洗去除了 52 条样本（主要是愤怒类），可能反而减少了模型学习愤怒特征的机会，导致愤怒→悲伤的错分更严重。

---

## 7. 下一步建议

1. **回查清洗逻辑**：检查 `split_train_cleaned.csv` 去除了哪些样本，特别是愤怒/尴尬类是否被过度清洗
2. **不对称 dropout**：RoBERTa dropout 降至 0.1-0.2，保持 BiLSTM/Classifier 的 0.3
3. **数据增强**：对愤怒/尴尬/惊讶等小样本类别做同义词替换/回译增强
4. **Label Smoothing**：添加 eps=0.1，降低模型对"悲伤"的过度自信
5. **尝试悲伤下采样**：将悲伤类从 33% 降至 20%，强制模型学习其他类别特征
